## How to Contribute

### Reporting Issues

- Check existing issues before creating a new one
- Include device model, Android version, and steps to reproduce
- Attach logs if relevant (sanitize any sensitive data)

### Code Contributions

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Test thoroughly
5. Submit a pull request

### Documentation

Documentation improvements are always welcome:
- Fix typos or clarify confusing sections
- Add examples or diagrams
- Translate to other languages

## Code Style

- Follow existing code style in each directory
- Kotlin: Follow Android Kotlin style guide
- C: Follow kernel style for native code
- Shell: Use shellcheck-clean bash

## What NOT to Submit

**Never submit:**
- Private keys or signing material
- +codes or urbit identity secrets
- Real BootPackages with actual identities
- Pier data from real ships
- ROM signing keys or AVB keys
- Factory images or target_files zips
- Any personally identifiable information

See [SECURITY.md](SECURITY.md) for details.
